# デプロイメント管理

このプロジェクト専用のスキル。Firebase Hostingの公開・非公開管理とGitのコミット・プッシュを実行する。

## トリガー

以下のいずれかのフレーズが含まれる場合にこのスキルを適用する：
- 「公開して」
- 「非公開にして」
- 「更新して」

## コマンド

### 「公開して」の場合

Firebase Hostingを公開する：

```bash
firebase deploy --only hosting
```

### 「非公開にして」の場合

Firebase Hostingを無効化する：

```bash
firebase hosting:disable
```

### 「更新して」の場合

以下の手順を実行する：

1. **変更内容を確認**
   - `git status`で変更されたファイルを確認
   - `git diff`で変更内容を確認

2. **コミットメッセージを作成**
   - 変更内容を分析して適切なコミットメッセージを生成
   - 日本語または英語で、変更内容が明確に分かるメッセージを作成
   - 例：
     - `feat: ビザ情報の検索機能を追加`
     - `fix: モバイル表示のレイアウトを修正`
     - `update: コンテンツを更新`

3. **ステージング**
   ```bash
   git add .
   ```

4. **コミット**
   ```bash
   git commit -m "作成したコミットメッセージ"
   ```

5. **プッシュ**
   ```bash
   git push
   ```

## 注意事項

- すべてのコマンドはプロジェクトルートディレクトリで実行する（ワークスペースのルートディレクトリ）
- 「更新して」の場合、コミットメッセージは変更内容に基づいて適切に生成する
- エラーが発生した場合は、エラーメッセージを確認して適切に対処する
